Installation

For Visual Studio IDE, the laziest way is to look on nuget package manager. Search with keyword pepper.

Integration

Pepper must be integrated in a file where test runner looks at it in each time before closing the test cases.

Coded UI

Implementation must be done in method with [TestCleanup] attribute.

Selenium

Implementation must be done in method with [TearDown] attribute.

Specfow

Implementation must be done in method with [AfterScenarioBlock] attribute.

Text Logs

Present under Pepper namespace. Logs each activities during automation.

Logs.WriteLog(logs,"C:\\Pepper\\Logs");

  • logs:- Type of StringBuilder class. Give here all logs.
  • Directory:- Type of String. Type here a directory name where logs to be entered.
  • Note:- The Logs contains elapsed time, execution entry date, text executor machine name along with OS info.

SQL Logs

Present under Pepper namespace. Trace down each test activities into SQL Server.

Logs.WriteLogsInSql("Login", "Password", "Server",sb);

  • Login:- Type of String class. Give here SQl login name.
  • Password:- Type of String. Type here SQL password.
  • Server:- Type of String. Type here SQL Server name.
  • sb:- Type of StringBuilder class. Give here all logs.

Mongo Logs

Present under Pepper namespace. Keeps test activities records in Mongo DB.

Logs.WriteLogsInMongoDB("logs", "[connectionString]");

  • logs:- Type of StringBuilder class. Give here all logs.
  • connectionString:- Type of String. Type here connectionString.
  • Note:- ConnectionString is optional parameter, if not provided by user then create DB in local machine.
