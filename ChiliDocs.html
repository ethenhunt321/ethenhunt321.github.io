Installation

For Visual Studio IDE, the laziest way is to look on nuget package manager. Search with keyword chili.

Integration

Chili must be integrated in a file where test runner looks at it in last before closing the application.

Coded UI

Implementation must be done in method with [AssemblyCleanup] attribute.

Selenium

Implementation must be done in method with [AssemblyCleanup] attribute.

Specfow

Implementation must be done in method with [AfterTestRun] attribute.

NUnit

Implementation must be done in method with [OneTimeTearDown] attribute.

Outlook

Present under chili namespace. Auto trigger mail from configured outlook with visual.

Outlook.SendMailViaOutlook(To List, DataTable, "Subject/Title", "Your additional Text");

  • To List:- Type of List of string. Give here all recipeients name.
  • DataTable:- Type of DataTable. Give here all Feature,TestCase Names and Outcomes in order mentioned.
  • Subject/Title:- Type of string. Give here your subject name or title of mail.
  • Your additional Text:- Type of string Geive here your addition message which need to be sent.

Excel

Present under chili namespace. Gives an excel report with rich graphs.

ExcelReport.ExportExcelReport(DataTable, "Test Reports", "path",[featureList]);

  • path:- Type of string. Give here your directory name where report will be saved.
  • DataTable:- Type of DataTable. Give here all Feature,TestCase Names and Outcomes in order mentioned.
  • Test Reports:- Type of string. Give here your workbook name or sheet name of excel.
  • [featureList]:- Type of List of string. This is an optional parameter but its presence doesn't make any sence.

DHTML

Present under chili namespace. Gives a dynamic html report with rich graphs.

Binders.CreateChiliReport(DataTable, "Test Report", "path");

  • path:- Type of string. Give here your directory name where report will be saved.
  • DataTable:- Type of DataTable. Give here all Feature,TestCase Names and Outcomes in order mentioned.
  • Test Reports:- Type of string. Give here your workbook name or sheet name of excel.
  • Note:-In First overload it expects total,pass and fail count.But its presence doesn't make any sence.

JSON

Present under chili namespace. Gives a JSON report.

JSONReport.ExportJsonReport(DataTable, "path");

ExportJsonReport(DataTable);

  • path:- Type of string. Give here your directory name where report will be saved.
  • DataTable:- Type of DataTable. Give here all Feature,TestCase Names and Outcomes in order mentioned.
  • Note:-In First overload it expects just one parameter. Post using this overloaded method you can store result back to a string variable.
